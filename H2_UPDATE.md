# H2 Database Security Update

## Overview
Updated H2 database dependency from version 1.4.197 to 2.3.232 to address critical security vulnerabilities.

## Security Vulnerabilities Fixed
- **CVE-2022-23221** (CVSS 9.8): Remote code execution vulnerability in H2 Console
- **CVE-2021-42392**: Additional security vulnerability in H2 database
- **CVE-2022-45868**: Security vulnerability in H2 database  
- **CVE-2018-10054**: Security vulnerability in H2 database

## Changes Made

### 1. Dependency Update
- Updated H2 dependency version in `pom.xml` from `1.4.197` to `2.3.232`

### 2. SQL Syntax Updates for H2 2.x Compatibility
Updated SQL initialization scripts to use H2 2.x compatible syntax:

#### Identity Column Syntax
- **Before**: `INTEGER IDENTITY PRIMARY KEY`
- **After**: `INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`

Files updated:
- `src/main/resources/create_catalogue_book.sql`
- `src/main/resources/create_patron_db.sql`
- `src/main/resources/create_lending_book_db.sql`
- `src/main/resources/create_sheets_db.sql`

#### Sequence Usage Syntax
- **Before**: `sequence_name.nextval`
- **After**: `NEXT VALUE FOR sequence_name`

Files updated:
- `src/main/java/io/pillopl/library/catalogue/CatalogueDatabase.java`
- `src/main/java/io/pillopl/library/lending/book/infrastructure/BookDatabaseRepository.java`
- `src/main/java/io/pillopl/library/lending/dailysheet/infrastructure/SheetsReadModel.java`

## Compatibility Notes
- H2 2.x requires updated SQL syntax for identity columns and sequence usage
- The `GENERATED BY DEFAULT AS IDENTITY` syntax allows both auto-generation and manual assignment of ID values
- All existing database functionality remains intact after the syntax updates
- Spring Boot embedded database configuration remains unchanged
- H2 Console configuration in `application-dev.yml` remains compatible
